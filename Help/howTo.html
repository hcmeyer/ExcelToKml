HowTo use the ExcelToKml program

  1. Prepare Excel Data - Using a SpreadSheet program, prepare the Excel file with Name and Address (required) and other data (optional). The Column Data help item describes the column and data format required. Re-arrange the columns in your data to fit this.

  2. Read Excel Data - Start the ExcelToKml program, read the data in, using the Excel Open Menu command or Toolbar button. The Excel row data appears in the Top Left of the program window, in a Form.

  3. GeoCode the Data - Use the Kml GeoCode Check Menu Command or Toolbar button to verify that all rows on all sheets have valid Latitude and Longitude data. If any rows do not have valid Latitude and Longitude data, the check will stop at the first such row, and put a message in the status.

    1. Check in the Form that the row has enough Street, Town, State, Zip data to have a valid postal address. Press the Lookup pushbutton on the form to add Longitude and Latitiude data to the row. If the lookup succeeds, Update the row data, use the GeoCheck again, to continue checking the rows.

    2. If the Lookup fails, producing an error message, try to correct the Street, Town, State, Zip address data, and repeat the Lookup. If the row cannot be corrected, go to the Next row, and continue checking from there. A row with invalid or missing Latitude and Longitude will be skipped when generating Placemarks.

  4. Kml - After as many as possible of the rows have been GeoCoded, use the Kml Menu Command or Toolbar button to generate the Kml data. After the Kml data is generated, it can be reviewed in Top Right of the program window in the Tree widget.

  5. Save Data to Files - Use the Kml Save or Kml Save As Menu Commands or Toolbar Buttons to save the Kml data.

    If the Excel Data was modified by GeoCoding or other editing, use the Excel Save or Excel Save As Menu commands to save the Excel data

  6. Done - Quit the program.
