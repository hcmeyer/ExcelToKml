Other

  • Window Layout - The interior of the program window is divided into three regions.

    • Top Left is the Form, showing the row data from the Excel file.
    • Top Right is the Tree, showing the Kml data in a Tree Widget.
    • Bottom is a list of all status, warning and error messages issued by the program during this run.

    The regions are separated by splitter bars, shaped like an upside down letter "T". The splitter bars can be moved within the window by dragging with the mouse, changing the proportions of the window for each region.

  • Window Frame and Controls The program window is surrounded by an ordinary top level frame, and is resizable and closeable.

    The top right corner of the frame has the usual -, rectangle, and X controls, that minimize, maximize and un-maximize, and close the program window. Closing the program window with the X will act the same as the Quit, asking to save changed data.

    The program frame can be resized by dragging the frame edges and corners.

    The bottom of the frame is used as a status message display. Copies of all messages are saved in the bottom region of the program, in a scrollable list.

  • Save and Restore - The program saves several of its characteristics when closed, and restores them when next started. These are Window Frame Size and Position, the proportions of the splitters, and the last Excel File opened. The most recently open Excel File will be re-opened, if possible, when the program is started.

  • Printing - The ExcelToKml program does not have any printing functions.

    Decoding the printing options stored in the Excel file is very complex, error prone, and unnecessary. The Excel or other Spreadsheet program should be used to print this data.

    The Kml file is much simpler, it is an Xml text file. It can be viewed and printed with any text editor, such as Notepad. The Tree view in the Top Right of the program window is an abstract version of this data, showing structure and content.

  • Adding rows or sheets - The ExcelToKml does not allow adding rows or sheets to the Excel data.

    Similar to printing, decoding the options needed to store new rows or sheets in the Excel file is very complex, error prone, and unnecessary. The Excel or other Spreadsheet program should be used to add rows or sheets.

  • Editing, General - The Excel Data row editing and File Saving is designed to allow lookup of Latitude and Longitude data from Street, Town, State, Zip address data. It will also allow minor editing of other columns. Serious editing should be done with Excel or another Spreadsheet program before using this program.

    The Kml data is not editable because it is derived from the Excel data. Any changes in the Kml data would have to be reflected back to the Excel data, to allow it to be saved. Much too difficult. Any editing should be done on the Excel data, either with the Row editing of the ExcelToKml program, or an Excel or other Spreedsheet program.

  • mailto - A mailto is a type of URL. It contains an eMail address, and looks like
    "< a href="mailto:user@domain" > User Name " in the raw html. When it appears in a browser, you will see User Name, and when you click it, the browser will open your mail client program (Outlook, ThunderBird, or ...), and open a new message with the eMail address for composition and sending.

    This mailto will become part of the popup for the placemark. Clicking it in the popup will open the mail client, etc. These can be in the "Website" column in the Excel data. The original application for the ExcelToKml program had links to websites in this column. For a mailing list, a mailto with member's eMail addresses would allow easy eMail links to members to be shown.
