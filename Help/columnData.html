The ExcelToKml program requires the data in Excel files to have a particular format. The program uses data in the first thirteen columns of the Excel file. Any other columns in the file are not used. There may be more than one sheet in the Excel file, each sheet will be an internal folder in the Kml file, with the sheet name as a folder name.

The program can only Open and Save .xls files, the older binary format used by Excel. It cannot Open or Save the newer .xslx or .xml Excel file formats.

The first thirteen columns should formatted as simple text, with the labels, in the first row of the sheet:

  1. First Name
  2. Last Name
  3. Description
  4. Street
  5. Town
  6. State
  7. Zip
  8. County
  9. Telephone
  10. Email
  11. Website
  12. Latitude
  13. Longitude

These labels in the first row will be checked, to make sure the Excel File is correctly formatted, or the correct type of file is being read. If the labels are not correct, a warning message is issued.

The only data really required in each row is First and Last Name, and either Latitude and Longitude, or enough of Street, Town, State, Zip data to produce a postal address that can be looked up to produce Latitude and Longitude. The other data, if present, will be combined in the Popup for the PlaceMark.

Any row that does not have valid Latitude and Longitude will not produce a PlaceMark, and a warning message for that row. The Kml GeoCode Check in the program that will check rows for missing or invalid Latitude and Longitude.
